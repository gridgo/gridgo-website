Gridgo

Asynchronous, event-driven framework for distributed systems

Getting started

Star

Key Features

Non blocking

Everything asynchronous. Whether it is I/O connections or processing routines

Abstract I/O

Easily define I/O connections with an endpoint, supported HTTP server/client, Kafka, RabbitMQ, Redis and more.

Event-driven

Everything is message. You write code to handle the event and that's it

Install

              
<dependency>
    <groupId>io.gridgo</groupId>
    <artifactId>gridgo-core</artifactId>
    <version>x.x.x</version>
</dependency>
